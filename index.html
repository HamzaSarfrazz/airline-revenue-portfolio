Five-layer pipeline: from browser automation through to Google Sheets sync.
01
Browser automation layer
Playwright launches a persistent Chromium context with stealth mode. A JSON.parse hook intercepts flight batch payloads before the DOM renders them. Stall detection fires if zero data is received for 120s — browser auto-restarts, up to 3 retries per sector. A random 20–40s sleep between sectors mimics human browsing cadence.
playwrightplaywright-stealthchromiumasync_playwright
→
02
Normalization & parsing layer
Raw JSON batches are de-duplicated by hash. Each flight is evaluated for non-stop status via a 3-condition check (leg count, segment count, stop label). Departure times are normalized to 12h format across date-time strings, ISO timestamps, and raw HH:MM values. Only the cheapest fare per (route, flight_number) pair is retained.
pandasregexdatetimejson
→
03
Multi-currency handling
Each time slot (10am, 4pm, 6pm, 9pm) is independently configured with a target currency. The URL builder appends &currency= for non-PKR requests; the API returns pre-converted prices in the payload. Coverage: PKR, USD, GBP, AED, SAR across 40+ airport code pairs in Domestic, UAE, and KSA markets.
PKRUSDAEDSARGBPurllib.parse
→
04
Google Sheets sync
A CONFIG worksheet maps (route, flight_number) pairs to exact cell addresses. All fare and time updates are batched into a single batch_update() call to stay within Sheets API rate limits. Unmapped flights are logged separately for review. Timestamp cells update automatically on every push, giving RM staff a clear data-freshness indicator.
gspreadgoogle-oauth2batch_updateSheets API v4
→
05
Streamlit Cloud deployment
Deployed to Streamlit Cloud with Chromium installed at runtime via setup.sh. Secrets (GCP service account, spreadsheet IDs per slot) are stored in st.secrets. A persistent Queue bridges the background scraping thread to the Streamlit UI, enabling real-time log streaming and live data table updates without page refresh.

About this project

The Fare Intelligence Suite is a sanitized architectural blueprint modeled after real automation tools I've designed. In competitive airline markets, manual fare tracking causes severe reporting lags. This project simulates an end-to-end extraction and sync loop—proving how web scraping, fault-tolerant programming, and API integration can stream live competitive market data directly to commercial dashboards and spreadsheet matrices without manual effort.

All data and corporate targets in this public demonstration are strictly synthetic. The operational logic is real.
